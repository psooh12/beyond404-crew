# 크루앱 백엔드 연결 표시 수정 기록

## 작업 배경
- 크루앱 배포 화면에서 수거 요청 목록이 비어 있는데도 `수거 요청 목록을 불러오지 못했습니다`라는 문구가 보여 백엔드 연결 실패처럼 보였다.
- 실제 API `https://crew.swapit.my/api/crew/calls`는 `200`과 빈 배열 `[]`을 반환하고 있어, 백엔드 연결은 정상인 상태였다.

## 수정 내용
- 크루 요청 목록 화면에서 API 성공과 API 실패 상태를 분리했다.
- API 성공 후 빈 목록이면 `현재 수락 가능한 수거 요청이 없습니다`와 `백엔드 연결 정상`을 표시한다.
- API 실패 시에만 백엔드 연결 오류 안내를 표시한다.
- 크루 요청 API 호출에 `cache: "no-store"`를 추가해 새 요청 목록을 매번 다시 확인하도록 했다.

## 확인 결과
- `npm run build` 정상 통과.
- 로컬 크루앱 `http://localhost:3001`, 휴대폰 접속용 `http://172.30.1.55:3001` 응답 확인.

## 대화 메모
- 사용자는 크루앱에서 백엔드 연결이 잘 안되는 것처럼 보인다고 공유했다.
- 확인 결과 연결 자체는 정상이고, 빈 목록 상태를 오류처럼 보여주는 UI 상태 처리 문제로 판단했다.
